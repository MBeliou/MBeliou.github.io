Ionic Vue in 5 Minutes

Apr 22, 2019
2 min read

Since my article on VueJs, curiosity has brought me to look for a way to develop a mobile application with it.

After spending some time googling around and playing with a few frameworks, I ended up finding out about the release of Ionic 4 and it’s newfound support for Vue.

Why choose Ionic?

Ionic has been around since 2013 and helps you build hybrid mobile apps using web technologies, which simply means that after writing the app once, it will be deployable across Web, Android, IOS and Electron.

Being built on top of Cordova means that Ionic not only comes battery charged with all the usual plugins to native components, but also brings a set of reworked integrations from database to payments.

On a personal note, I’ve found that developing Vue apps using Ionic feels much more natural than with other implementations such as Nativescript, especially when handling re-rendering and list updates.

Still, all is not perfect when using Vue with Ionic:

Being a beta release obviously means that a few edges remain to be smoothed, chief among which the native plugins which should be introduced only through Capacitor if the web is among your targets.

I’ve also found that the docs are quite lacking regarding the CSS and the handling of the shadow DOM for styling. I highly encourage anyone encountering issues to watch this outstanding video.

My experience with Ionic Vue

As a way to try out Ionic Vue, I’ve made a simple cryptocurrency application which can be found on this repository.

I said earlier that using Ionic feels quite natural if you’ve already used Vue, everything mostly just works. You don’t need to make your own makeshift router, your own store implementation or your internationalization plugin, you can just use the dependencies you are used to.

Even though everything “just works”, I’d avise to be careful when dealing with the store, and explicitly use only deep copies.

Support for Vue is still sadly quite limited and you can really feel Ionic being centered on Angular despite their plans to provide additional support for React.

Time will tell

I am looking forward to the future of Vue using Ionic, using it felt pleasant despite encountering a few issues. I expect support to pick up when the framework reaches a more stable state.

I am planning on using it for a few personal projects and will update this post in due time with relevant informations.

Until then, please try this wonderful work of code for yourself. I promise you’ll enjoy it!
